**Отчет по Лабораторной работе 9**
----------------------------------------
----------------------------------------
**Тема: Гибкое моделирование. Вариант: Частная клиника**
-------------------------------------

Пользовательская история:

"Как администратор клиники, я хочу найти медицинскую карту пациента или создать новую, если карта отсутствует, чтобы записать пациента на прием к врачу."
      
Критерии приемки
------------------------------

Для пользовательской истории были сформулированы следующие критерии приемки:

* Система позволяет найти медицинскую карту пациента по ФИО и номеру телефона
* Если медицинская карта пациента отсутствует, администратор может создать новую карту
* Система отображает список врачей по выбранной специализации
* Система показывает свободные временные слоты для записи
* Запись на прием создается только при отсутствии временных конфликтов

Роли в системе:
------------------------------

 * Пациент - инициирует обращение в клинику, но не взаимодействует с системой напрямую.
 * Администратор - выполняет поиск и создание медицинских карт, а также оформляет запись на прием.
 * Врач - работает с медицинскими картами пациентов и планами лечения.

Карта пользовательской истории
------------------------------
------------------------------

Основная активность:
------------------------------

Запись пациента на прием

Подзадачи:
------------------------------

* Поиск медицинской карты пациента
* Создание медицинской карты (при отсутствии)
* Выбор специализации врача
* Выбор врача
* Выбор даты и свободного временного слота
* Подтверждение записи на прием

**MVP (минимально жизнеспособный продукт)**:
------------------------------

* Поиск медицинской карты
* Создание медицинской карты
* Отображение свободных слотов
* Создание записи на прием

Диаграмма классов (упрощённая):
------------------------------

* Пациент

**Эскизы интерфейса**:
------------------------------

В рамках работы были разработаны низкодетализированные эскизы интерфейса системы, отражающие основной пользовательский сценарий.

Описание экранов:
------------------------------

1. **Экран поиска медицинской карты.** Используется для поиска пациента по ФИО и номеру телефона. При отсутствии карты система предлагает создать новую.
2. **Экран создания медицинской карты.** Предназначен для ввода персональных данных пациента и создания медицинской карты.
4. **Экран записи на прием.** Позволяет выбрать специализацию врача, врача, дату и свободный временной слот.
5. **Экран подтверждения записи.** Отображает полную информацию о записи и позволяет подтвердить или отменить ее создание.
